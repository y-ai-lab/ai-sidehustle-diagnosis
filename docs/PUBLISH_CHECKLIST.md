# 公開前チェックリスト

「現実派AI副業診断」を公開する前に確認する項目です。
外部サービス連携や本番URL設定を入れる前に、まずMVPとして安全に見せられる状態かを確認します。

## 必須チェック

- `npm.cmd run build` が成功する
- `npm.cmd test` が成功する
- build/testは並列ではなく、上から順番に実行する
- スマホ幅でホーム、質問、結果ページが横スクロールしない
- 10問すべて回答して結果ページまで進める
- 結果ページのCTAがURL未設定時に準備中表示になる
- Xシェアボタンが開ける

## 文言チェック

- 「誰でも簡単に稼げる」などの煽り表現がない
- 月1万円を現実的に目指すトーンになっている
- AI副業初心者が次の行動を理解できる
- 各タイプの弱みや注意点も書かれている
- 著作権、商標、確認不足の情報発信を軽く注意できている

## データチェック

- 質問は `src/data/questions.ts` にまとまっている
- 結果文は `src/data/results.ts` にまとまっている
- CTAは `src/data/cta.ts` にまとまっている
- 30日ロードマップは `src/data/roadmaps.ts` にまとまっている
- Xシェア文と公開URLは `src/data/share.ts` にまとまっている
- コンポーネント内に質問文、結果文、アフィリエイトリンクを直書きしていない

## 公開URL決定後にやること

- 詳細手順は [URL_SETUP_GUIDE.md](URL_SETUP_GUIDE.md) を確認する
- `src/data/share.ts` の `siteUrl` に公開URLを入れる
- CTAの遷移先が決まった場合だけ `src/data/cta.ts` の `url` を入れる
- 本番URLを入れた後に `npm.cmd run build` と `npm.cmd test` を実行する
- 実URL導線を入れた場合は、ユーザー確認が必要な変更として報告する

## 公開後に見ること

- どの結果タイプが多いか
- どのCTA文言が読まれていそうか
- 質問で迷いそうな表現がないか
- スマホで長すぎるセクションがないか
